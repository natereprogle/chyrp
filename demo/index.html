Install

Install from npm. The CSS is a separate import so you can bundle it however you like.

npm install chyrp
import { chyrp } from 'chyrp';
import 'chyrp/style.css';
CDN: Drop in the IIFE bundle and access the library at window.Chyrp: 
<link rel="stylesheet" href="https://unpkg.com/chyrp/dist/style.css" />
<script src="https://unpkg.com/chyrp/dist/index.iife.js"></script>
<script>Chyrp.chyrp.info('Hello'); </script>

Quick start

Show a toast with a single call. Every form returns a ToastHandle you can use to dismiss or update later.

chyrp({ body: 'Hello world' });

Styles

Four built-in visual styles, each with a default icon and accent color. Use the convenience methods or pass style to the main function.

chyrp.info('Saved');
chyrp.warning('Almost out of space');
chyrp.error('Upload failed');
chyrp.loading('Working…');

// equivalent:
chyrp({ body: 'Saved', style: 'info' });

Title & body

Add a bold heading above the body for two-line layouts.

chyrp({
  title: 'Saved',
  body: 'Your changes are live.',
  style: 'info',
});

Position

Six anchor positions. Mobile viewports always force bottom-center for thumb reach.

chyrp.info('Hello', { position: 'top-right' });

Timing

The default timeout is 4 seconds. Set timeout to a custom number of milliseconds, 0 to disable, or use persistent: true as a shorthand for the latter.

chyrp.info('Quick',     { timeout: 1500 });
chyrp.info('Default');                          // 4000 ms
chyrp.info('Patient',   { timeout: 8000 });
chyrp.info('Forever',   { persistent: true }); // user must dismiss

Debounce

Identical toasts fired within the debounce window are suppressed and the existing handle is returned. Useful for chatty event sources. Click the button rapidly:

// Default debounce window is 100 ms; bump it for noisier sources:
chyrp.info('Same message', { debounce: 2000 });

Channels

Tag toasts under a channel name and dismiss them as a group. The channel renders as a small italic label in the toast's corner.

import { chyrp, dismissChannel } from 'chyrp';

chyrp.info('Connection lost', { channel: 'network', persistent: true });
chyrp.info('Retrying…',       { channel: 'network' });

// later:
dismissChannel('network');

Custom icons

Override the default icon with a string (text or emoji), an HTMLElement (cloned per toast), or false to hide the icon column entirely.

chyrp.info('Party time',  { icon: '🎉' });

const svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg');
// …populate svg…
chyrp.info('Custom DOM',  { icon: svg });

chyrp.info('No icon box', { icon: false });

Action buttons

Add buttons under the body. Returning false from onClick keeps the toast open; otherwise it auto-dismisses after the click.

chyrp({
  title: 'File deleted',
  body: 'document.pdf was moved to trash',
  persistent: true,
  actions: [
    { label: 'Undo', style: 'primary', onClick: () => restore() },
    { label: 'Dismiss' },
  ],
});

Promise integration

Wrap a promise to show a loading toast that converts to success or error when it settles. Each branch may be a string, an options object, or a function returning either.

chyrp.promise(api.saveUser(user), {
  loading: 'Saving…',
  success: (user) => `Saved ${user.name}`,
  error:   (err)  => ({ title: 'Save failed', body: err.message }),
});

Determinate progress

For loading toasts with known total work, pass max and value. The icon becomes a donut filled in proportion to value / max. Update via handle.update({ value }).

const handle = chyrp.loading('Uploading', { max: 100, value: 0 });

uploader.on('progress', (n) => handle.update({ value: n }));
uploader.on('done',     ()  => handle.update({
  style: 'info', body: 'Upload complete', timeout: 2000,
}));

Updating in place

handle.update(opts) mutates a live toast — change body, style, icon, actions, timeout, anything. Useful for streaming status updates without flapping the UI.

const h = chyrp.loading('Step 1/3: connecting');
setTimeout(() => h.update({ body: 'Step 2/3: uploading' }), 700);
setTimeout(() => h.update({ body: 'Step 3/3: verifying' }), 1400);
setTimeout(() => h.update({ style: 'info', body: 'Done!', timeout: 2000 }), 2100);

Sound

Chyrp toasts demand attention. Pass a named chime, a custom URL, or true for a style-aware default. Named chimes are synthesized via Web Audio — no asset dependency. Sound is opt-out; the global default is style-aware.

chyrp.info('Saved',  { sound: 'success' });    // named chime
chyrp.error('Save failed', { sound: true });   // style default → 'error'
chyrp.info('Custom', { sound: '/ding.mp3' });  // your own audio asset

Custom chime notes

Pass a ChimeNote[] array directly to play a fully custom synthesized chime on a single toast:

chyrp.info('Custom sound!', {
  sound: [
    { frequency: 880,   startOffset: 0,    duration: 0.1, amplitude: 0.2 },
    { frequency: 1108,  startOffset: 0.08, duration: 0.15, amplitude: 0.18 },
    { frequency: 1318,  startOffset: 0.16, duration: 0.2, amplitude: 0.15 },
  ],
});

Overriding preset sounds

Use configure({ soundPresets }) to globally replace any of the four built-in chime presets (success, error, alert, gentle). Omitted presets keep their defaults.

import { configure } from 'chyrp';

configure({
  soundPresets: {
    success: [
      { frequency: 880, startOffset: 0, duration: 0.12, amplitude: 0.2 },
      { frequency: 1320, startOffset: 0.1, duration: 0.18, amplitude: 0.18 },
    ],
    // error, alert, gentle keep their defaults
  },
});

Opt-out of sound

Sound isn't for everyone. Use configure({ sound: false }) to globally disable all sounds.

import { configure } from 'chyrp';

configure({
  sound: false
});

Pause-on-hover

The auto-dismiss timer pauses while the toast is hovered or keyboard-focused. The progress bar freezes and resumes from where it left off. Auto-disabled on touch-primary devices.

chyrp.info('Read me carefully',  { pauseOnHover: true,  timeout: 5000 });
chyrp.info('Slipping away…',     { pauseOnHover: false, timeout: 5000 });

Swipe-to-dismiss

Drag a toast horizontally (or vertically for centered positions) past a threshold to dismiss it with momentum. Disable with swipe: false.

chyrp.info('Drag me out',     { persistent: true });
chyrp.info('Pinned in place', { swipe: false, persistent: true });

Dismiss helpers

Tear down all live toasts at once, or just those tagged with a channel.

import { dismissAll, dismissChannel } from 'chyrp';

dismissAll();
dismissChannel('network');

Global configuration

Set defaults once with configure(). Per-call options always win.

configure({
  position: 'bottom-right',
  pauseOnHover: true,
  sound: 'gentle',
});

alert() override

Opt-in helper that replaces window.alert with a toast-based version. Returns a function that restores the native alert.

import { interceptAlert } from 'chyrp';

const restore = interceptAlert();
window.alert('hello'); // shows as an info toast

restore(); // put the native alert back

Accessibility

Toasts are role="button" with tabindex="0", so they appear in the tab order. Keyboard focus pauses the auto-dismiss timer (just like hover) — handy for screen-reader users who need extra time. Action buttons inside toasts are real <button>s.

  • Tab — focus a live toast
  • Enter / Space — dismiss the focused toast
  • Tab into the overflow pill, then Enter — dismiss the oldest hidden toast

Mobile

Below 600 px viewport width, toasts automatically reposition to the bottom edge regardless of position, expand to full width with safe-area inset padding, and use larger touch targets and progress bar. Pause-on-hover is disabled on touch-primary devices since hover doesn't apply.

Resize this page to under 600 px wide to see the mobile layout, or use your browser's device toolbar.

Reduced motion

When prefers-reduced-motion: reduce is set, all transform-based entry/exit animations collapse to a fade, the spinner slows down, and the overflow pill skips its slide-in. Toasts still appear and dismiss — they just don't fly.

Toggle Reduce motion in your OS settings (or DevTools → Rendering → Emulate CSS media feature) to verify.

Dark mode

Toasts automatically adapt to dark backgrounds via prefers-color-scheme: dark. All colors are defined as CSS custom properties prefixed with --tt-, so you can override any of them.

To force a specific theme regardless of system preference, add a light or dark class to the <html> element:

<!-- Force dark mode -->
<html class="dark">

<!-- Force light mode -->
<html class="light">

<!-- Follow system preference (default) -->
<html>
Use the toggle in the top-right corner of this page to preview each mode. The .dark class applies dark variables unconditionally, while .light prevents the prefers-color-scheme media query from activating.

Customizing colors

Override any --tt-* variable to theme the toasts. For example:

:root {
  --tt-bg: #1a1a2e;
  --tt-title-color: #eee;
  --tt-body-color: #ccc;
  --tt-info: #00d2ff;
}
